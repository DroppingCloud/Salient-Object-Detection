\section{摘要}

本报告围绕显著性目标检测任务展开，以 ResNet-18 \cite{he2016deep} 为骨干网络，
系统研究了基于深度学习的显著图预测方法。

\bigskip
\noindent\textbf{关键词：}显著性目标检测；ResNet-18；编解码器；注意力机制；多尺度融合

%----------------------------------------------------------------
\section{引言}

显著性目标检测（Salient Object Detection, SOD）旨在从自然图像中定位并分割出最吸引人类视觉注意力的前景区域，
输出与输入图像等尺寸的二值或软性预测掩码（mask）。该任务在图像检索、视频理解、自动驾驶等领域具有广泛的应用价值。

传统方法主要依赖于在手工设计的特征下进行，难以适应复杂图像场景和结构，泛化能力有限。
随着深度学习的发展，基于卷积神经网络的方法在语义特征提取上取得了显著进步，成为当前的主流范式。

我们的贡献总结如下：

\begin{enumerate}
  \item 实现了两种基于 ResNet-18 的基线方法（FCN 解码器与 U-Net 跳跃连接解码器）并进行对比分析；
  \item 提出了融合边缘增强分支、通道注意力与多尺度特征融合的改进网络（ResSOD-Improved）；
  \item 设计了结合二值交叉熵（BCE）与结构相似性（SSIM）的混合损失函数；
  \item 在 DUTS 数据集上进行了系统的消融实验，定量验证了各模块的贡献。
\end{enumerate}

\subsection{ResNet}

%----------------------------------------------------------------
\subsubsection{基本原理}

ResNet\cite{he2016deep} 是深度卷积神经网络中的经典骨干结构，其核心思想是通过残差连接缓解深层网络训练中的退化问题。
与传统卷积网络直接学习目标映射 $H(x)$ 不同，残差网络学习的是输入与输出之间的残差部分 $F(x)=H(x)-x$。这种设计使得网络在加深时更
容易学习恒等映射，从而降低深层网络的优化难度。

残差块（Residual Block）是 ResNet 的基本组成单元（图 \ref{fig:residual_block}），主要由残差分支和捷径分支两部分构成。残差分支用于对输入特征进行非线性变换，捷径分支则将输入特征直接传递到输出端，并与残差分支的输出进行逐元素相加。

\begin{figure}[t]
  \centering
  \includegraphics[width=0.7\textwidth]{fig/resnet1.png}
  \caption{Residual block}
  \label{fig:residual_block}
\end{figure}

%----------------------------------------------------------------
\subsubsection{网络结构}

ResNet 通常由一个初始卷积层和若干个残差阶段组成（图~\ref{fig:resnet}）。以 ImageNet 版 ResNet 为例，
输入图像先经过 7×7 conv + BN + ReLU + maxpool 得到低层特征，然后依次经历 layer1、layer2、layer3、layer4 四个残差阶段。

每个阶段由多个残差块组成，空间分辨率逐步下降，通道数逐步增加。一般情况下，ResNet18/34 使用 BasicBlock，ResNet50/101 使用更深的 Bottleneck Block。

\begin{figure}[t]
  \centering
  \includegraphics[width=0.7\textwidth]{fig/resnet3.png}
  \caption{ResNet 网络结构}
  \label{fig:resnet}
\end{figure}

%----------------------------------------------------------------
\subsubsection{基于 ResNet 的显著目标检测网络}

显著目标检测（Salient Object Detection, SOD）旨在从图像中预测最具视觉吸引力的前景区域。由于该任务需要生成像素级显著图，因此网络既需要高层语义信息来定位显著目标，又需要低层空间信息来恢复目标边界。

基于 ResNet 的显著目标检测网络通常采用编码器-解码器（Encoder-Decoder）结构。其中 ResNet 作为编码器提取多尺度特征，
解码器负责融合不同层级特征并生成显著图。

\begin{align*}
    S = D(C_1, C_2, C_3, C_4)
\end{align*}

其中 $C1, C2, C3, C4$ 表示解码器输出的各层级特征，$D(\cdot)$ 表示解码器，$S$ 表示输出显著图。

% ----------------------------------------------------------------
\subsection{PoolNet}

% ----------------------------------------------------------------
\subsubsection{基本思想}

PoolNet \cite{liu2019simple}是一种基于编码器—解码器结构的显著目标检测网络（图~\ref{fig:poolnet}），其核心思想是充分融合深层语义信息与浅层空间细节信息，从而生成区域完整、边界清晰的显著性预测图。
PoolNet 的核心创新点包括：

\begin{itemize}
    \item \textbf{全局上下文增强}：通过顶层特征的多尺度池化，使网络感受野扩大，可区分局部高对比度背景与真正显著目标。
    \item \textbf{多尺度特征融合}：通过自顶向下的解码过程将高层语义信息与浅层空间信息融合，增强浅层特征的表达能力并抑制噪声。
\end{itemize}

\begin{figure}[t]
  \centering
  \includegraphics[width=1.0\textwidth]{fig/poolnet.png}
  \caption{PoolNet 模型架构}
  \label{fig:poolnet}
\end{figure}

% ----------------------------------------------------------------
\subsubsection{金字塔池化模块（PPM）}

PoolNet 引入金字塔池化模块（Pyramid Pooling Module，PPM）用于捕获多尺度上下文信息。
对于深层输入特征 \( X_{\text{deep}} \in \mathbb{R}^{B \times C \times H/32 \times W/32} \) ，首先经过多尺度自适应平均池化形成多尺度全局上下文特征：
\[
X_{s} = \text{Upsample}\Big(\text{AvgPool}_{s}(X_{\text{deep}})\Big)
\]
其中 \(s \in \{1,3,5\}\) 表示不同池化输出尺寸，Upsample 将池化特征上采样回深层特征分辨率。然后再与原始深层特征进行拼接增强：
\[
X_{\text{PPM}} = \text{Conv}\Big([X_{\text{deep}}, X_{1}, X_{3}, X_{5}]\Big)
\]

PPM 输出随后上采样到浅层特征尺寸，用作 Global Guiding Flow (GGF)，为解码器提供全局语义指导。

% ----------------------------------------------------------------
\subsubsection{深度池化特征融合（Deep Pooling Layer）}

在解码阶段，PoolNet 采用自顶向下融合的策略，核心模块是深度池化层（Deep Pooling Layer）。其处理流程为：

\begin{enumerate}
    \item 高层特征 \(F_{\text{high}}\) 上采样至当前层特征分辨率，
    并与当前层特征 \(F_{\text{skip}}\) 及 GGF 全局信号特征 \(F_{\text{info}}\) 相加。
    \[
    F_{\text{merge}} = F_{\text{high}}^{\uparrow} + F_{\text{skip}} + F_{\text{info}}
    \]
    
    \item 对融合后的特征执行多尺度平均池化（Adaptive Average Pooling），最后每个池化分支通过卷积和 ReLU 后上采样回原始分辨率，
    融合不同尺度的上下文信息，从而增强感受野。
    \[
    F_{\text{pool}} = F_{\text{merge}} + \sum_{s \in \{2,4,8\}} \text{Upsample}\Big(\text{AvgPool}_{s}(F_{\text{merge}})\Big)
    \]
    
    \item 对融合后的特征进行卷积整合。
    \[
    F_{\text{out}} = F_{3\times3}(F_{\text{pool}})
    \]
\end{enumerate}

这种自顶向下的融合策略保证了浅层特征同时拥有高分辨率空间信息和深层语义信息，
提升显著图对边缘和细节的敏感性，同时抑制浅层噪声干扰。

% ----------------------------------------------------------------
\subsection{GateNet}

% ----------------------------------------------------------------
\subsubsection{基本思想}

GateNet \cite{zhao2020suppress} 是一种基于门控机制的显著目标检测网络（图 \ref{fig:gatenet}），
核心思想是在编码器与解码器之间引入门控单元，自适应控制不同层级特征向解码器传递的信息量。
GateNet 的核心创新点包括：

\begin{itemize}
    \item \textbf{特征筛选}：通过门控权重抑制非显著区域响应，减少背景干扰。
    \item \textbf{层级平衡}：根据不同编码层对预测结果的贡献，动态调整特征融合强度。
    \item \textbf{双分支互补}：FPN 分支负责主体区域预测，Parallel 分支补充边界和细节信息。
    \item \textbf{多尺度增强}：利用 Fold-ASPP 增强深层特征的上下文表达能力。
\end{itemize}

\begin{figure}[t]
  \centering
  \includegraphics[width=1.0\textwidth]{fig/gatenet.png}
  \caption{GateNet 模型架构}
  \label{fig:gatenet}
\end{figure}

% ----------------------------------------------------------------
\subsubsection{Fold-ASPP}

对于深层语义特征，GateNet 引入 Fold-ASPP 模块以增强多尺度上下文建模能力。普通 ASPP 通过不同空洞率的卷积扩大感受野，但较大的空洞率会导致采样点之间相关性减弱，不利于细节结构建模。
而 Fold-ASPP 通过 Fold 操作将局部邻域折叠到通道维度，使空洞卷积作用于局部区域而非孤立采样点，从而在扩大感受野的同时保留局部结构连续性（图~\ref{fig:fold}）。

Fold-ASPP 采用多条并行分支提取不同尺度的上下文信息，多尺度分支输出融合后，为后续解码器提供更强的高级语义指导。

\begin{figure}[t]
  \centering
  \includegraphics[width=0.7\textwidth]{fig/fold-aspp.png}
  \caption{普通空洞卷积与折叠空洞卷积对比}
  \label{fig:fold}
\end{figure}

% ----------------------------------------------------------------
\subsubsection{Gate Units}

传统 FPN 或 U-Net 通常直接将编码器特征传递给解码器，但这种 all-pass skip connection 可能将背景噪声和误导性上下文一并引入解码过程。
为此，GateNet 设置门控单元（Gate Units）显式控制编码器特征向解码器传递的信息量。

对于第 \(i\) 层，门控单元同时接收当前编码器特征 \(E_i\) 和上一层解码器特征 \(D_{i+1}\)，并生成门控权重：
\[
G_i =
P\Big(
\sigma(
\text{Conv}([E_i, D_{i+1}])
)
\Big)
\]
其中 \([\,\cdot\,]\) 表示通道拼接，\(\sigma(\cdot)\) 为 Sigmoid 函数，\(P(\cdot)\) 表示全局平均池化。

对于最高层，由于不存在更高层解码器特征，门控权重由编码器特征和对应的过渡层特征共同生成：
\[
G_i =
P\Big(
\sigma(
\text{Conv}([E_i, T_i])
)
\Big)
\]

门控单元输出两个权重 $G_i = [G_i^{1}, G_i^{2}]$，
其中 \(G_i^{1}\) 作用于 FPN 分支，控制当前层特征参与逐级解码的强度；
\(G_i^{2}\) 作用于 Parallel 分支，用于选择补充残差信息的编码器特征。

% ----------------------------------------------------------------
\subsubsection{FPN 分支}

FPN 分支负责生成主要的显著性预测，其解码过程采用自顶向下融合策略：高层语义特征逐级上采样，并与当前层经过门控筛选的编码器特征融合。

\[
D_i =
\text{Conv}
\Big(
G_i^{1} \cdot T_i + \text{Up}(D_{i+1})
\Big)
\]

在这种机制下，高层特征提供全局语义，低层特征补充空间细节，门控单元则在二者之间起到抑制背景干扰、平衡层级贡献的作用。

% ----------------------------------------------------------------
\subsubsection{Parellel 分支}

逐级解码结构可能导致部分细节信息在自顶向下融合过程中被削弱，GateNet 为此增加了并行分支，将各层经过第二组门控权重筛选后的特征直接汇聚起来，用于补充 FPN 分支遗漏的信息。

\[
F_{\text{cat}} =
\text{Cat}
\Big(
D_1,
\text{Up}(G_1^{2} \cdot T_1),
\text{Up}(G_2^{2} \cdot T_2),
\cdots,
\text{Up}(G_n^{2} \cdot T_n)
\Big)
\]

随后通过卷积生成残差预测，并与 FPN 分支结果相加。
\[
S_{\text{final}}
=
\text{Conv}(F_{\text{cat}}) + D_1
\]

其中 \(D_1\) 表示 FPN 分支得到的初步显著性预测。该残差结构使并行分支不必重新预测完整显著图，而是专注于补充边界、细长结构和局部细节。

从功能上看，FPN 分支更依赖高层语义信息，用于确定显著目标主体；Parellel 分支更强调低层细节信息，用于恢复局部结构。二者通过门控机制形成互补。

% \begin{align*}

%   Recall&=\frac{|M \cap G|}{|G|}\\

% 其中，$M$ 表示预测显著图二值化结果，$G$ 表示真实标注图。在实际评价中，通过改变二值化阈值，

%----------------------------------------------------------------
\subsubsection{MAE}

MAE（Mean Absolute Error）表示平均绝对误差，用于衡量预测显著图与真实标注图之间的像素级差异。其定义为：
\begin{equation*}
  MAE=\frac{1}{W\times H}\sum_{x=1}^W\sum_{y=1}^H|S(x,y)-G(x,y)|
\end{equation*}

其中，$S$ 表示预测显著图，$G$ 表示真实标注图。MAE 直接在连续显著图上进行计算，
不依赖于二值化阈值。因此，MAE 能够反映预测图与真实标注图在整体像素层面的接近程
度。MAE 数值越小，说明模型生成的显著图与真实标注越接近，预测误差越低。

%----------------------------------------------------------------
\subsubsection{F-measure}

F-measure 是 Precision 和 Recall 的加权调和平均，用于综合衡量模型的准确性和完整性。其定义为：
\begin{equation*}
  F_{\beta}=\frac{(1+\beta^2)\cdot Precision \cdot Recall}{\beta^2\cdot Precision + Recall}
\end{equation*}

其中，$\beta$ 用于调节 Precision 和 Recall 的相对重要性，通常设置 $\beta^2=0.3$。
F-measure 数值越大，说明模型在显著目标区域检测的准确性和完整性方面表现越好。

%----------------------------------------------------------------
\subsubsection{Weighted F-measure}

Weighted F-measure（$F_{\beta}^{w}$）是传统 F-score 的加权扩展形式，用于更加准确地评价显著性检测结果。普通 F-score 主要依据像素级 Precision 和 Recall 计算模型性能，但对预测误差的空间位置和结构影响考虑不足。为此，$F_{\beta}^{w}$ 引入加权机制，对不同区域的预测误差赋予不同权重，从而能够更好地反映显著目标的区域完整性、边界质量以及背景抑制效果。
其定义为：
\begin{equation*}
  F_{\beta}^{w}
  =
  \frac{(1+\beta^2)\cdot Precision^{w}\cdot Recall^{w}}
  {\beta^2\cdot Precision^{w}+Recall^{w}}
\end{equation*}

其中，$Precision^{w}$ 表示加权准确率，$Recall^{w}$ 表示加权召回率，$\beta$ 用于调节二者的相对重要性。$F_{\beta}^{w}$ 数值越大，说明预测显著图与真实标注图在像素分布和空间结构上越一致，模型的显著性检测性能越好。

%----------------------------------------------------------------
\subsubsection{E-measure}

E-measure（Enhanced-alignment Measure）是一种综合考虑全局统计信息与局部像素匹配关系的显著性检测评价指标，用于衡量预测显著图与真实标注图之间的整体对齐程度和结构一致性。其定义为：
\begin{equation*}
  E_{\phi}=\frac{1}{W\times H}\sum_{x=1}^{W}\sum_{y=1}^{H}\phi(x,y)
\end{equation*}

其中，$\phi(x,y)$ 表示增强对齐矩阵在像素位置 $(x,y)$ 处的取值，$W$ 和 $H$ 分别表示图像的宽度和高度。E-measure 在评价过程中不仅关注预测结果与真实标注之间的像素级差异，还引入了全局均值信息来刻画二者在整体区域分布上的一致性。因此，相比仅基于像素分类结果的评价指标，E-measure 能够更全面地反映显著目标区域的完整性、前景与背景的区分能力以及预测结果的结构质量。
E-measure 越大表示预测显著图与真实标注图之间的一致性越高，模型的显著性检测效果越好。

\subsection{实验细节}

本项目基于 PyTorch 深度学习框架实现，使用 NVIDIA RTX 4090 GPU 完成模型训练与测试。
为保证不同模型之间的公平比较，各实验均采用 ImageNet 预训练的 ResNet-34 作为骨干网络，
其余新建卷积层与融合层采用随机初始化。实验默认使用 DUTS-TR 作为训练集，并在 ECSSD 数据集上进行测试与性能评估。

在数据预处理阶段，输入图像首先被缩放至 $256 \times 256$，随后随机裁剪为 $224 \times 224$ 作为网络输入。
训练阶段采用随机水平翻转进行数据增强，同时使用ImageNet 预训练模型的统计量对输入图像进行归一化。
显著性标注图的二值化阈值设置为 $0.5$。

优化器采用 AdamW，权重衰减系数设为 $5\times10^{-4}$。
为避免预训练骨干网络参数发生过大扰动，实验对骨干网络与新建层采用不同学习率：
骨干网络学习率设为 $3\times10^{-5}$，其余新建参数学习率设为 $2\times10^{-4}$。
所有模型均训练 25 个 epoch。学习率调度采用 MultiStep 策略，
在第 15 个和第 22 个 epoch 将当前学习率衰减为原来的 $10\%$。

训练损失由显著性检测中常用的像素级分类损失与结构约束损失共同组成。
对于主输出，采用 BCE、SSIM Loss 与 Soft IoU Loss 的加权组合：
\[
\mathcal{L}_{main}
=
\mathcal{L}_{BCE}
+
\mathcal{L}_{SSIM}
+
\left(1-\operatorname{IoU}_{soft}\right).
\]
其中 Soft IoU 定义为
\[
\operatorname{IoU}_{soft}
=
\frac{\sum_i p_i g_i}
{\sum_i (p_i + g_i - p_i g_i) + \epsilon}
\]
$p_i$ 表示预测显著图在第 $i$ 个像素处的概率值，$g_i$ 表示对应的真实标注值，
$\epsilon$ 为防止除零的极小常数。

对于具有多尺度输出或深监督分支的模型，除主输出损失外，还对辅助输出施加 BCE 与 Soft IoU 约束：
\[
\mathcal{L}_{aux}
=
\mathcal{L}_{BCE}
+
\left(1-\operatorname{IoU}_{soft}\right).
\]
最终训练目标写作：
\[
\mathcal{L}
=
\mathcal{L}_{main}
+
\lambda \mathcal{L}_{aux},
\]
其中辅助损失权重设为 $\lambda=0.4$。对于仅包含单一输出的模型，则仅计算主输出损失。
